4. Ошибок нет. Есть 2 причины появления ошибок:
- Пересекающаяся сеть (при наложении сетей или при вложении одной сети в другую)

Практическая часть:
После установки всех необходимых утилит мы создаём Vagrantfile, каталог под файлы Ansible и пишем их.
Посел написание поднимаем ВМ:
vagrant up
И наблюдаем исполнение playbook.
Проверка доступности интернета с любого сервера:
vagrant ssh office1Server
ping 8.8.8.8
traceroute 8.8.8.8
Проверка связности между всеми серверами:
ping 192.168.0.2   # centralServer
ping 192.168.1.2   # office2Server
Проверка, что дефолтный маршрут через eth0 отключён:
ip route show
sudo reboot
# после перезагрузки проверить iptables на inetRouter:
vagrant ssh inetRouter
sudo iptables-save
Если всё работает, то домашнее задание выполнено!
